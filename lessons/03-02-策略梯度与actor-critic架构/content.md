# 3.2 策略梯度与Actor-Critic架构

## 课程概述

本课时深入探讨策略梯度方法与Actor-Critic架构，这是强化学习领域解决连续动作空间和高维动作空间问题的核心框架，也是现代大模型对齐技术（PPO、GRPO）的理论基础。相较于上一章的基于价值函数的方法，策略梯度方法直接对策略进行参数化优化，避免了离散动作空间枚举和连续空间价值函数插值的困难。通过引入Actor-Critic双网络架构，我们可以在保持策略优化的同时，利用价值函数估计来降低梯度方差，加速模型收敛。理解这些基础概念，对于掌握大模型RLHF中的PPO算法至关重要。

**学习目标**
- 理解价值基础方法的局限性，掌握其在连续/大离散动作空间中的困境
- 掌握策略梯度定理的数学推导，理解∇_θ J(θ) = E[∇_θ log π_θ(a|s) * Q^{π_θ}(s,a)]的核心含义
- 理解REINFORCE算法的Monte Carlo策略梯度估计方法
- 认识策略梯度的方差问题及其对收敛的影响
- 掌握Actor-Critic架构的核心思想：Actor（策略网络）与Critic（价值网络）的协同机制

- 掌握GAE（广义优势估计）的偏差-方差权衡原理
- 理解Actor-Critic架构与PPO/GRPO在大模型对齐中的联系

**前置知识**：强化学习基础概念（MDP、折扣因子、价值函数）、神经网络基础、概率论基本知识

## 1. 价值基础方法的局限性

### 1.1 连续动作空间的困境

Q-learning及其衍生算法（如DQN）在处理连续动作空间时面临根本性困难。回顾Q-learning的核心公式：

```
Q(s,a) ← Q(s,a) + α[r + γ max_a' Q(s',a') - Q(s,a)]
```

**核心问题**：`max_a' Q(s',a')` 操作要求我们在每个状态下枚举所有可能的动作，选择Q值最大的动作。对于连续动作空间，这意味着需要求解一个优化问题：

```
max_a' Q(s',a') = max_a' f(s', a')
```

其中 f(s', a') 是一个连续函数。这类优化问题通常计算代价高昂，且难以保证全局最优解。

**示例**：假设一个机械臂控制任务，动作空间是关节角度的连续取值（每个关节 -180°到180°）。在每个状态下计算 max_a' Q(s',a') 需要：
- 对动作空间进行离散化采样
- 多次前向传播计算Q值
- 求解优化问题找到近似最大值

这导致推理阶段计算成本极高，严重影响实时控制能力。

### 1.2 大离散动作空间的问题

当动作空间规模巨大时（如大语言模型的词表大小通常为数万到数十万），基于价值函数的方法同样面临严峻挑战：

**存储问题**：Q表或Q网络需要存储每个状态-动作对的Q值。对于词表大小V=32000、状态表示维度D=4096的LLM，完整Q表的参数量为 4096 × 32000 ≈ 1.3亿。

**探索效率**：在巨大动作空间中，ε-greedy探索策略效率极低。以V=32000为例，即使ε=0.05，智能体在95%的时间选择"看起来最好"的动作，但实际最优动作可能从未被充分探索。

**最优Q值逼近**：价值网络需要精确逼近每个动作的Q值，这是个极其困难的高维回归问题。随着动作空间维度增加，逼近误差累积，导致策略质量下降。

```
┌─────────────────────────────────────────────────────────┐
│         价值基础方法的适用性与局限性                      │
├─────────────────────────────────────────────────────────┤
│  适用场景                                                │
│  ✓ 动作空间小（离散且可枚举）                            │
│  ✓ 环境可精确建模（已知转移概率）                         │
│  ✓ 样本效率要求高（需要off-policy学习）                  │
├─────────────────────────────────────────────────────────┤
│  局限性                                                 │
│  ✗ 连续动作空间：max操作需要求解优化问题                 │
│  ✗ 大离散动作空间：存储和逼近困难                       │
│  ✗ 策略表达受限：间接通过价值函数导出策略                │
└─────────────────────────────────────────────────────────┘
```

## 2. 策略梯度定理

### 2.1 从价值最大化到策略最大化

策略梯度方法换了一个思路：不通过价值函数间接导出策略，而是直接参数化策略函数：

```
π_θ(a|s) = P(a|s; θ)
```

策略网络接收状态输入，输出动作的概率分布。优化目标是最大化期望回报：

```
J(θ) = E_{τ~π_θ}[R(τ)]
```

其中 τ = (s_0, a_0, s_1, a_1, ..., s_T) 是一条完整的轨迹（trajectory），R(τ) 是该轨迹的累积奖励。

**核心优势**：
- 连续动作空间：直接输出动作分布，无需枚举
- 大离散动作空间：策略网络作为参数化函数，共享参数而非存储完整表格
- 随机策略：天然输出随机策略，有利于探索

### 2.2 策略梯度定理的数学推导

策略梯度定理是策略梯度方法的理论基石。设 J(θ) = E_τ~π_θ[R(τ)]，我们对参数 θ 求梯度：

**第一步：轨迹概率分解**

轨迹 τ 的概率可以分解为：

```
P(τ|θ) = π_θ(a_0|s_0) * P(s_1|s_0,a_0) * π_θ(a_1|s_1) * ...
```

**第二步：对数梯度技巧**

利用 log 的导数性质：

```
∇_θ P(τ|θ) = P(τ|θ) * ∇_θ log P(τ|θ)
```

**第三步：轨迹回报梯度**

```
∇_θ J(θ) = ∇_θ E_τ~π_θ[R(τ)]
          = E_τ~π_θ[R(τ) * ∇_θ log P(τ|θ)]
```

**第四步：轨迹分解**

```
∇_θ log P(τ|θ) = Σ_{t=0}^{T-1} ∇_θ log π_θ(a_t|s_t)
```

**最终定理**：

```
∇_θ J(θ) = E_τ~π_θ[Σ_{t=0}^{T-1} ∇_θ log π_θ(a_t|s_t) * R(τ)]
```

进一步利用因果性（Causality）：t时刻的动作只依赖于t时刻之后的奖励，可以简化为：

```
∇_θ J(θ) = E_τ~π_θ[Σ_{t=0}^{T-1} ∇_θ log π_θ(a_t|s_t) * G_t]
```

其中 G_t = r_t + γr_{t+1} + γ²r_{t+2} + ... 是从时刻t开始的折扣回报。

### 2.3 更通用的形式

策略梯度定理的更通用形式使用状态-动作对的重要性加权：

```
∇_θ J(θ) = E_{s~d^{π_θ}, a~π_θ(·|s)}[∇_θ log π_θ(a|s) * Q^{π_θ}(s,a)]
```

其中：
- d^{π_θ}(s) 是策略 π_θ 下的状态访问分布（state visitation distribution）
- Q^{π_θ}(s,a) 是策略 π_θ 下的动作价值函数

**物理直觉**：这个公式告诉我们如何调整策略参数使期望回报最大化。对于每个状态-动作对：
- 如果 Q(s,a) > 0（动作表现优于平均），则增加 π(a|s) 的概率
- 如果 Q(s,a) < 0（动作表现差于平均），则减少 π(a|s) 的概率

**与神经网络的联系**：∇_θ log π_θ(a|s) 是策略网络的对数概率梯度。如果策略网络是神经网络，我们可以用标准的反向传播计算这个梯度。

## 3. REINFORCE算法

### 3.1 Monte Carlo策略梯度估计

REINFORCE是最基础的策略梯度算法，采用Monte Carlo方法估计期望梯度：

```
∇_θ J(θ) ≈ (1/N) Σ_{i=1}^{N} Σ_{t=0}^{T-1} ∇_θ log π_θ(a_{i,t}|s_{i,t}) * G_{i,t}
```

其中 N 是采样的轨迹数量，G_{i,t} 是第i条轨迹在时刻t的折扣回报。

**算法流程**：
1. 使用当前策略 π_θ 采样 N 条轨迹
2. 对每条轨迹的每个时间步计算折扣回报 G_t
3. 计算策略梯度估计
4. 使用梯度上升更新参数

```
REINFORCE算法伪代码：

输入：初始策略参数 θ
输出：优化后的策略参数 θ

1. 重复以下步骤直到收敛：
2.   使用 π_θ 采样轨迹 {τ_i}, i=1,...,N
3.   对每条轨迹，计算 G_t = Σ_{k=t}^{T} γ^{k-t} r_k
4.   计算梯度估计：∇_θ J ≈ (1/N) Σ_i Σ_t ∇_θ log π_θ(a_{i,t}|s_{i,t}) * G_{i,t}
5.   更新参数：θ ← θ + α ∇_θ J
```

### 3.2 REINFORCE的实际计算

以PyTorch为例，REINFORCE的实际计算如下：

```python
# 策略网络
policy_net = PolicyNetwork(state_dim, action_dim)
optimizer = torch.optim.Adam(policy_net.parameters(), lr=0.001)

# 收集一条轨迹
states, actions, rewards = [], [], []
state = env.reset()
done = False

while not done:
    state_tensor = torch.FloatTensor(state)
    probs = policy_net(state_tensor)  # 动作概率分布
    dist = torch.distributions.Categorical(probs)
    action = dist.sample()  # 采样动作
    
    next_state, reward, done = env.step(action.item())
    
    states.append(state_tensor)
    actions.append(action)
    rewards.append(reward)
    
    state = next_state

# 计算折扣回报
discounted_rewards = []
G = 0
for r in reversed(rewards):
    G = r + gamma * G
    discounted_rewards.insert(0, G)

discounted_rewards = torch.FloatTensor(discounted_rewards)

# 计算策略梯度
log_probs = []
for state, action in zip(states, actions):
    probs = policy_net(state)
    dist = torch.distributions.Categorical(probs)
    log_probs.append(dist.log_prob(action))

log_probs = torch.stack(log_probs)

# 策略梯度：∇_θ log π(a|s) * G
policy_loss = -(log_probs * discounted_rewards).mean()

optimizer.zero_grad()
policy_loss.backward()
optimizer.step()
```

### 3.3 方差问题：REINFORCE的核心挑战

REINFORCE面临严峻的方差问题。高方差梯度估计导致：
- 收敛速度慢，需要大量采样
- 训练不稳定，容易出现剧烈波动
- 样本效率低

**方差来源**：使用整条轨迹的回报 G_t 作为策略梯度的基准则。G_t 包含从当前时刻到轨迹结束的所有随机性：
- 状态转移的随机性（环境不确定性）
- 未来动作选择的随机性（策略随机性）

**直观理解**：在滑雪游戏中，如果我们用一个完整赛季的总得分来评价某个转弯动作的好坏：
- 总得分受很多后续动作影响
- 即使这个转弯很好，如果后面的动作差，总得分也会低
- 反之亦然——这个转弯不好，但后面的动作弥补了

这导致对单个动作的评估充满噪声，需要大量轨迹平均才能获得可靠的梯度估计。

**对比分析**：

| 评估方式 | 方差 | 偏差 | 示例 |
|---------|------|------|------|
| 完整轨迹回报 G_t | 高 | 低 | 用总得分评价每个转弯 |
| 基线减均值 | 中 | 低 | 用相对得分评价 |
| 优势函数 A(s,a) | 低-中 | 中 | 用相对于平均水平的优势评价 |

## 4. Actor-Critic架构

### 4.1 核心思想：双网络协同

Actor-Critic架构通过引入两个网络解决方差问题：

**Actor（策略网络）**：
- 输入：状态 s
- 输出：动作概率分布 π(a|s; θ)
- 作用：负责选择动作（"演员"）

**Critic（价值网络）**：
- 输入：状态 s（或状态-动作对）
- 输出：价值估计 V(s; ω) 或 Q(s,a; ω)
- 作用：评估当前状态/动作的好坏（"评论家"）

```
┌────────────────────────────────────────────────────────┐
│              Actor-Critic 架构                         │
│                                                        │
│         状态 s                                         │
│            │                                          │
│            ├───→ [Actor Network] ──→ 动作概率分布 π(a|s)│
│            │                                           │
│            └───→ [Critic Network] ──→ 价值估计 V(s)     │
│                                                        │
│                 ↑                                      │
│                 │ 优势 A(s,a) = Q(s,a) - V(s)           │
│                 │ 指导策略更新                          │
└────────────────────────────────────────────────────────┘
```

### 4.2 Critic的作用：降低方差

Critic的核心贡献是提供一个**基线（baseline）**，将高方差的回报估计替换为低方差的**优势估计**。

**基线减均值法**：

原始策略梯度使用 G_t 作为评估：
```
∇_θ J ≈ ∇_θ log π(a|s) * G_t
```

引入状态价值基线 V(s)：
```
∇_θ J ≈ ∇_θ log π(a|s) * (G_t - V(s))
```

**关键洞察**：E[G_t|s] = V(s)，因此 E[V(s)|s] = V(s)，基线项的期望为0，不引入偏差。但基线可以显著降低方差，因为 G_t 和 V(s) 的相关性通常很高。

**优势函数**：更进一步，我们不只用状态价值作为基线，而是用优势函数：

```
A(s,a) = Q(s,a) - V(s)
```

优势函数表示在状态s下执行动作a相对于平均水平的优势：
- A(s,a) > 0：动作a优于平均水平，应该增加其概率
- A(s,a) < 0：动作a差于平均水平，应该减少其概率

**梯度公式**：

```
∇_θ J(θ) = E[∇_θ log π_θ(a|s) * A(s,a)]
```

用优势函数替换原始回报，无需整个轨迹的信息，可以用单步TD误差近似：

```
A(s,a) ≈ r + γV(s') - V(s)  (TD误差)
```

### 4.3 Actor-Critic算法流程

```
Actor-Critic算法伪代码：

初始化：
  - 策略网络参数 θ（Actor）
  - 价值网络参数 ω（Critic）

重复以下步骤直到收敛：
1. 使用当前策略 π_θ 采样轨迹
2. 对每个(s,a,r,s')计算TD误差：
   δ_t = r_t + γ V(s_t+1; ω) - V(s_t; ω)
3. 计算优势估计（可选使用GAE，见下一节）：
   A_t = δ_t + γλ δ_t+1 + ... (GAE)
4. 更新Critic（价值网络）：
   ω ← ω + β_1 * ∇_ω Σ A_t²  （最小化价值估计误差）
5. 更新Actor（策略网络）：
   θ ← θ + β_2 * ∇_θ Σ log π_θ(a_t|s_t) * A_t （策略梯度上升）
```

### 4.4 双网络更新的物理直觉

**Critic更新**：Critic试图准确估计状态价值，损失函数是最小化优势函数的平方期望：

```
L(ω) = E[(Q(s,a) - V(s;ω))²]
```

当Critic准确时，优势函数期望为0，意味着策略在当前状态下选择的动作的平均优势为0。这是合理的——策略应该调整到使得所有可选动作的优势期望为0（即策略已经是最优或局部最优）。

**Actor更新**：Actor利用Critic提供的优势信息进行策略梯度更新：

```
θ ← θ + α * E[∇_θ log π_θ(a|s) * A(s,a)]
```

如果 A(s,a) > 0，策略梯度指向增加该动作的概率方向；如果 A(s,a) < 0，则相反。

## 5. 广义优势估计（GAE）

### 5.1 偏差-方差权衡

优势函数的估计面临经典的偏差-方差权衡：

**TD(1)（即Monte Carlo）**：
```
A_t = G_t - V(s_t)
```
- 无偏但高方差
- 需要完整轨迹采样

**TD(0)**：
```
A_t = r_t + γV(s_{t+1}) - V(s_t) = δ_t
```
- 低方差但有偏
- 单步TD误差

### 5.2 GAE：调和方案

GAE通过指数加权平均组合多步TD误差，实现偏差-方差权衡：

```
A_t^{GAE(γ,λ)} = (1-λ)(δ_t + γλδ_{t+1} + γ²λ²δ_{t+2} + ...)
               = Σ_{l=0}^{∞} (γλ)^l δ_{t+l}
```

其中：
- γ 是折扣因子（控制远期奖励的重要性）
- λ 是GAE参数（控制偏差-方差权衡）
  - λ → 0：接近TD(0)，低方差高偏差
  - λ → 1：接近Monte Carlo，无偏高方差

**物理直觉**：GAE(γ, λ) 是无限多步TD误差的指数加权平均，权重以 (γλ)^l 衰减。λ 控制了多远的多步TD误差被纳入优势估计。

**实际应用中的典型值**：
- λ = 0.95~0.98（接近Monte Carlo但仍有较低方差）
- γ = 0.99（关注长期回报）

**推导优势**：

当 λ = 1 时，GAE退化为Monte Carlo回报与当前价值估计的差：
```
A_t = G_t - V(s_t)
```

当 λ = 0 时，GAE退化为单步TD误差：
```
A_t = δ_t = r_t + γV(s_{t+1}) - V(s_t)
```

中间值提供平滑的偏差-方差曲线，允许根据具体任务调优。

## 6. Actor-Critic与LLM对齐

### 6.1 从游戏到语言模型

在传统强化学习场景（游戏、机器人控制）中：
- 状态：游戏画面、传感器读数
- 动作：离散的控制指令
- 奖励：游戏得分、任务完成度

在大语言模型对齐中：
- 状态：当前的词序列（prompt + response）
- 动作：下一个词的选择（词表中的词，V通常为32000+）
- 奖励：人类偏好、任务完成度、安全性等

**关键差异**：
1. **动作空间规模**：LLM的词表规模达到数万，远超传统强化学习任务
2. **序列决策**：每个词的选择都会影响后续的词序列，形成一个极长的动作序列
3. **奖励获取延迟**：整个回答生成完成后才能获得完整奖励

### 6.2 PPO中的Actor-Critic结构

PPO（Proximal Policy Optimization）是现代RLHF的基石算法，其核心就是Actor-Critic架构：

**Actor（策略网络）**：
- 策略网络 π_θ(a|s) 是语言模型本身
- 输入：词序列（状态）
- 输出：下一个词的概率分布
- 目标：最大化期望回报

**Critic（价值网络）**：
- 价值网络 V_ω(s) 估计状态价值
- 输入：词序列
- 输出：该序列的期望回报估计
- 作用：计算优势函数，降低梯度方差

**PPO损失函数**：

```
L^{CLIP}(θ) = E_t[min(r_t(θ)A_t, clip(r_t(θ), 1-ε, 1+ε)A_t)]

其中：
- r_t(θ) = π_θ(a_t|s_t) / π_{θ_old}(a_t|s_t)  是策略比率
- A_t 是优势函数估计
- clip 操作限制策略更新幅度
```

### 6.3 GRPO的革新

GRPO（Group Relative Policy Optimization）是PPO的变体，在LLM对齐场景下更加高效：

**PPO的问题**：
- 需要单独维护一个Critic网络（价值网络）
- Critic需要预训练或联合训练
- 计算开销大

**GRPO的核心思想**：
- 不使用绝对价值估计，而是使用组内相对比较
- 对同一个prompt采样多个响应
- 通过比较相对好坏来估计优势

```
A_i = (r_i - μ_r) / σ_r
```

其中 r_i 是第i个响应的奖励，μ_r 和 σ_r 是该组响应的均值和标准差。

**GRPO的优势**：
- 无需Critic网络，降低内存和计算开销
- 利用组内基线自然降低方差
- 更适合LLM场景（一个prompt可以生成多个response）

### 6.4 实践中的Critic初始化

在大模型RLHF实践中，Critic（价值网络）的初始化是一个关键问题：

**从头训练的困境**：
- 价值网络需要准确估计期望回报
- 但在训练初期，策略网络和价值网络都是随机的
- 随机初始化的Critic给出的优势估计不可靠

**常见解决方案**：

1. **奖励模型初始化**：
   - 使用Reward Model作为Critic的初始化
   - 奖励模型已经学习过人类偏好，可以提供有意义的价值估计
   - 这是ChatGPT训练流程中采用的方法

2. **预训练的价值头**：
   - 在策略网络（语言模型）上添加一个价值头（Value Head）
   - 价值头与策略网络共享大部分参数
   - 只训练少量额外参数

3. **混合训练策略**：
   - 初期使用较小的学习率，让Critic慢慢适应
   - 使用更大的折扣因子γ，使Critic关注长期回报
   - 采用warm-up策略，前N步只更新Critic

**实践建议**（基于DeepSeek-R1等开源模型的观察）：
- Critic与Actor使用相同的主干网络（语言模型）
- 价值头通常是一个线性层，投影到一维标量
- 训练早期价值估计不准确，优势函数可能被噪声主导
- 使用较大的λ值（0.95~0.98）增加稳定性

## 7. 算法对比总结

| 算法 | 策略表示 | 价值估计 | 方差控制 | 适用场景 |
|------|---------|----------|---------|---------|
| Q-Learning | 离散表格/网络 | 精确Q值 | 低 | 小离散动作空间 |
| DQN | 网络 | 目标网络 | 中 | 中等离散动作空间 |
| REINFORCE | 随机策略网络 | Monte Carlo回报 | 高 | 连续/大离散动作空间 |
| Actor-Critic | 随机策略网络 | 学习到的价值函数 | 中-低 | 各类动作空间 |
| PPO | 裁剪策略网络 | Critic网络 | 低 | LLM RLHF |
| GRPO | 裁剪策略网络 | 组内相对比较 | 低 | LLM RLHF（无需Critic）|

## 本章小结

1. **价值基础方法的局限**：在连续动作空间需求解优化问题，大离散动作空间存储和逼近困难

2. **策略梯度定理**：∇_θ J(θ) = E[∇_θ log π_θ(a|s) * Q^{π_θ}(s,a)]，直接优化策略而非价值函数

3. **REINFORCE算法**：Monte Carlo策略梯度估计，实现简单但方差高

4. **方差问题**：使用完整轨迹回报导致高方差，需要引入基线或优势函数

5. **Actor-Critic架构**：Actor负责策略输出，Critic负责价值估计，两者协同降低方差

6. **优势函数**：A(s,a) = Q(s,a) - V(s)，评估动作相对于平均水平的优劣

7. **GAE**：通过指数加权多步TD误差实现偏差-方差的灵活权衡

8. **LLM对齐应用**：PPO/GRPO本质上是Actor-Critic架构在超大词表序列决策问题上的应用
