## 1. GRPO的提出背景

### 1.1 PPO在大模型场景下的局限性

PPO（Proximal Policy Optimization）作为RLHF的标准算法，在中小规模强化学习任务中表现优异。然而，当应用于大语言模型的RLHF微调时，面临三个核心挑战：

**高方差问题**：大模型生成的长序列（如多步推理）仅有最终结果可被奖励模型评分。稀疏的奖励信号导致优势估计方差大，策略更新不稳定。

**架构臃肿问题**：PPO的标准实现需要同时运行四个模型副本——策略模型（Policy）、旧策略模型（Old Policy）、奖励模型（Reward）和价值模型（Critic）。对于数百亿参数的模型，仅模型权重就需要数GB到数十GB内存，加上梯度存储和优化器状态，GPU显存很快成为瓶颈。

**训练不稳定问题**：Value网络与策略网络之间的参数不一致可能导致优势估计偏差，累积误差会导致训练崩溃。

### 1.2 GRPO的核心洞察

DeepSeek团队观察到：对于大语言模型，可以利用**组内相对比较**来替代独立的Value网络进行优势估计。具体而言，对于每个输入问题q，生成G个不同的响应{o₁, o₂, ..., oG}，通过组内标准化计算每个响应的相对优势。这种设计蕴含一个关键洞察——**相对比较比绝对估计更稳定可靠**。

```
┌─────────────────────────────────────────────────────────┐
│              GRPO的核心设计理念                          │
├─────────────────────────────────────────────────────────┤
│  观察：                                                   │
│  - 人类难以给出绝对评分，但容易进行相对比较               │
│  - 同一问题的多个回答天然形成比较基准                     │
│                                                         │
│  设计：                                                   │
│  - 用组内统计量（均值/标准差）替代Value网络               │
│  - 每个回答的优势由其在组内的相对排名决定                │
│                                                         │
│  优势：                                                   │
│  - 无需训练Value网络，节省两个模型的显存                  │
│  - 组内标准化自动消除问题难度差异的影响                  │
│  - 方差降低，训练更稳定                                   │
└─────────────────────────────────────────────────────────┘
```

## 2. 组计算机制详解

### 2.1 组计算的核心原理

组计算（Group Computation）是GRPO的核心创新，它用组内统计基线替代了传统Actor-Critic架构中的Value网络。

**基线减法原理**：在强化学习中，从奖励中减去一个基线可以减少方差而不改变期望梯度。传统方法学习一个Value网络作为基线；GRPO则使用同组其他样本的统计量作为动态基线。

**核心公式**：
```
Â_i = (r_i - μ_group) / σ_group
```

其中：
- Â_i 是第i个响应相对于组内平均的优势
- r_i 是第i个响应的原始奖励
- μ_group 是组内所有奖励的均值
- σ_group 是组内所有奖励的标准差

### 2.2 组计算的工作流程

以数学推理问题为例，说明组计算的具体流程：

**Step 1: 多样化采样**
```
问题q: "求方程 x² - 5x + 6 = 0 的根"

生成G=4个响应：
o₁: "使用求根公式：x = (5±√(25-24))/2 = (5±1)/2，所以 x₁=3, x₂=2"
o₂: "因式分解：(x-2)(x-3)=0，所以 x=2 或 x=3"
o₃: "尝试 x=2：4-10+6=0，正确，所以 x=2"
o₄: "根据韦达定理..."
```

**Step 2: 奖励评估**
```
奖励模型评分：
r₁ = 9.5  （完整且正确的解答）
r₂ = 9.0  （正确但略简略）
r₃ = 7.0  （只找到一个根）
r₄ = 5.0  （方法不完整）
```

**Step 3: 组内标准化**
```
μ = (9.5 + 9.0 + 7.0 + 5.0) / 4 = 7.625
σ = √[((9.5-7.625)² + (9.0-7.625)² + (7.0-7.625)² + (5.0-7.625)²) / 4] ≈ 1.66

标准化优势：
Â₁ = (9.5 - 7.625) / 1.66 ≈ +1.13
Â₂ = (9.0 - 7.625) / 1.66 ≈ +0.83
Â₃ = (7.0 - 7.625) / 1.66 ≈ -0.38
Â₄ = (5.0 - 7.625) / 1.66 ≈ -1.58
```

**关键洞察**：组内标准化的优势在于自动适应不同难度的问题。简单问题可能所有回答都获得高奖励，困难问题可能所有回答都获得低奖励，但标准化后的优势值都能正确反映相对质量。

### 2.3 为什么组计算能替代Value网络

传统Actor-Critic架构中，Value网络的作用是提供状态价值估计V(s)作为优势计算的基线：Â(s,a) = Q(s,a) - V(s)。

GRPO通过组计算提供了另一种思路：
- **空间换时间**：用同组样本的统计量代替时间展开的估计
- **相对优于绝对**：估计相对排名比估计绝对价值更稳定
- **无需额外训练**：不需要反向传播训练Value网络参数

```
┌─────────────────────────────────────────────────────────┐
│         组计算 vs Value网络：方差减少的对比              │
├─────────────────────────────────────────────────────────┤
│  传统PPO（Value网络）：                                  │
│  - V(s) 通过神经网络估计，可能有偏差                     │
│  - 偏差累积导致优势估计不准确                            │
│  - 训练不稳定                                           │
│                                                         │
│  GRPO（组计算）：                                        │
│  - 用同组样本均值作为动态基线                             │
│  - 基线随样本自动调整，无参数误差                         │
│  - 方差随G增大而减小                                     │
└─────────────────────────────────────────────────────────┘
```

### 2.4 组大小G的选择

G（每组生成的回答数量）是GRPO的关键超参数：

| G值 | 特点 | 适用场景 |
|-----|------|----------|
| G = 4 | 最小配置，计算快 | 快速实验、推理速度优先 |
| G = 8 | 平衡配置 | 通用场景，默认推荐 |
| G = 16 | 稳定配置 | 追求训练稳定性 |
| G = 64 | 高精度配置 | 追求最优性能，计算资源充足 |

统计理论表明，方差随样本量增加而减小，但存在边际收益递减。实践中G=8或G=16是性价比最优的选择。

## 3. 双层KL控制机制

### 3.1 为什么需要双层控制

大语言模型的RLHF训练面临一个根本矛盾：一方面需要通过强化学习让模型学习新能力（可能需要较大的策略更新），另一方面需要防止模型遗忘预训练学到的知识和能力（需要限制策略更新幅度）。

单层KL控制难以同时满足这两个需求，因此GRPO设计了双层KL控制机制：

**微观KL控制**：在每个样本的奖励计算层面施加KL约束，影响单次学习信号强度
**宏观KL控制**：在整体策略更新目标函数层面施加KL约束，影响参数更新方向

### 3.2 微观KL控制（奖励层面）

微观KL控制在奖励计算阶段对每个响应单独施加惩罚：

```
r_t = r_φ(q, o_{≤t}) - β · D_KL(π_θ(·|q, o_{<t}) || π_ref(·|q, o_{<t}))
```

物理含义：对于每个生成的token，如果当前策略π_θ选择该token的概率与参考策略π_ref差异越大，则从即时奖励中扣除更多惩罚。

**直观理解**：假设生成"今天的天气真好"这个响应：
- 参考策略生成这句话的概率适中
- 当前策略生成这句话的概率极高（偏离参考）
- 微观KL控制会惩罚这种偏离，确保策略不会完全丢弃原有能力

### 3.3 宏观KL控制（策略更新层面）

宏观KL控制在策略更新目标函数中对整体策略施加约束：

```
J_GRPO(θ) = E[...min(ratio·Â, clip(ratio)·Â)] - β · D_KL(π_θ || π_ref)
```

第二项-β·D_KL(π_θ || π_ref)是宏观KL控制项，直接在目标函数中惩罚策略与参考策略的整体偏离。

**直观理解**：即使微观KL控制允许某些样本变化较大，宏观KL控制仍确保整个模型的参数更新不会过度偏离参考模型。

### 3.4 双层KL控制的协同作用

```
│                                                         │

│         │                      │                        │

│         │                      │                        │
│         │                      ▼                        │
```

### 3.5 参考策略正则化的重要性

参考策略π_ref通常是监督微调（SFT）后的基础模型，冻结其参数作为"能力锚点"。参考策略正则化之所以重要，基于以下原因：

**防止灾难性遗忘**：强化学习优化奖赏信号时，可能导致模型完全放弃预训练学到的知识和语言能力。KL惩罚确保策略不会过度偏离参考模型。

**维持基本能力**：模型在RLHF阶段通常只训练少数几个epoch，足够的KL约束能确保模型始终保持作为语言模型的基本能力。

**稳定训练过程**：参考模型提供了一个稳定的参照点，使策略优化过程更加平滑。

## 4. GRPO目标函数详解

### 4.1 完整的目标函数

GRPO的目标函数为：

```
J_GRPO(θ) = E[q~P(Q), {o_i}_{i=1}^G ~ π_θ_old(O|q)]
           · (1/G) Σ_{i=1}^G (1/|o_i|) Σ_{t=1}^{|o_i|}
             [ min( r_i,t(θ)·Â_i,t, clip(r_i,t(θ), 1-ε, 1+ε)·Â_i,t )
               - β · D_KL(π_θ || π_ref) ]
```

其中：
- q 是输入问题
- {o_i}_{i=1}^G 是G个采样响应
- r_i,t(θ) = π_θ(o_i,t|q, o_i,<t) / π_θ_old(o_i,t|q, o_i,<t) 是策略比率
- Â_i,t 是组计算的优势值
- ε 是裁剪参数（通常0.2）
- β 是KL惩罚系数
- D_KL(π_θ || π_ref) 是与参考策略的KL散度

### 4.2 与PPO目标函数的对比

```
┌─────────────────────────────────────────────────────────┐
│              PPO vs GRPO 目标函数对比                    │
├─────────────────────────────────────────────────────────┤
│  PPO：                                                   │
│  J_PPO(θ) = E[min(r_t·A_t, clip(r_t)·A_t)]              │
│                                                         │
│  GRPO：                                                  │
│  J_GRPO(θ) = E[min(r_i·Â_i, clip(r_i)·Â_i)] - β·KL     │
│                                                         │
│  关键差异：                                              │
│  1. PPO使用Value网络估计的优势A_t；                      │
│     GRPO使用组计算的优势Â_i                              │
│  2. PPO无显式KL项；GRPO显式包含与参考策略的KL惩罚        │
│  3. GRPO在组层面操作，PPO在单样本层面操作                │
└─────────────────────────────────────────────────────────┘
```

### 4.3 优势计算详解

GRPO中的优势值Â_i通过两步计算：

**第一步：奖励调整（微观KL控制）**
```
r_i' = r_φ(q, o_i) - β · D_KL(π_θ(·|q) || π_ref(·|q))
```

**第二步：组内标准化**
```
Â_i = (r_i' - mean(r')) / std(r')
```

### 5.2 计算效率对比

| 特性 | PPO | GRPO |
|------|-----|------|
| 模型副本数 | 4个 | 2个 |
| 需要训练的模型 | Policy + Critic | Policy |
| Value网络前向传播 | 需要 | 不需要 |
| 组计算开销 | 无 | G次奖励评估 |
| 内存占用 | 高（4个模型） | 低（2个模型） |
| 超参数复杂度 | 高（ε, λ, β等） | 中等（G, β, ε） |

### 5.3 内存效率的实际影响

以700亿参数模型为例：
- 单个模型权重：约14GB（FP16）
- 4模型副本PPO：仅权重就需要56GB，加上梯度（28GB）和优化器状态（112GB），总计约200GB
- 2模型副本GRPO：权重28GB，梯度14GB，优化器状态56GB，总计约100GB

这意味着在同样的GPU显存下，GRPO可以使用更大的批量大小，从而提高训练效率。

**PPO信息流**：


**GRPO信息流**：
```
问题q → Policy生成{G个响应} → Reward评分{各响应奖励} → 组计算优势Â → 策略更新（含KL约束）
```

## 6. GRPO在DeepSeek-R1中的应用

### 6.1 DeepSeek-R1的技术路线

DeepSeek团队使用GRPO作为DeepSeek-R1和DeepSeek-R1-Zero推理模型训练的核心算法。这两个模型在数学推理、代码生成和逻辑推理等任务上展现了卓越的性能，证明了GRPO在大规模训练中的有效性。

### 6.2 GRPO的关键配置

根据DeepSeek-R1技术报告，GRPO的关键配置为：

| 超参数 | 典型值 | 说明 |
|--------|--------|------|
| G（组大小） | 16-64 | 根据任务难度调整 |
| β（KL系数） | 0.01-0.05 | 控制策略偏离程度 |
| ε（裁剪参数） | 0.2 | 与PPO一致 |

### 6.3 GRPO的变体：结果监督与过程监督

GRPO支持两种监督模式：

**结果监督（Outcome Supervision）**：仅在完整响应结束时给予奖励，组内所有token共享同一优势值。适用于答案质量易于整体评估的任务。

**过程监督（Process Supervision）**：在推理过程的每个关键步骤给予奖励，需要额外的过程奖励模型。适用于需要正确推理步骤的复杂任务（如数学证明）。

## 本章小结

1. **GRPO的提出背景**：针对PPO在大模型RLHF中的高内存消耗、训练不稳定问题，DeepSeek提出用组计算替代Value网络

2. **组计算机制**：对每个问题生成G个响应，通过组内标准化计算优势Â_i = (r_i - μ) / σ，无需训练Value网络

3. **双层KL控制**：微观层在奖励计算时施加KL惩罚（每个响应），宏观层在策略更新目标函数中施加KL惩罚（整体策略）

4. **参考策略正则化**：防止灾难性遗忘，维持预训练能力，需要在"学习新能力"和"保持原有能力"之间平衡

5. **架构对比**：PPO需要4个模型副本（Policy、Old Policy、Reward、Critic），GRPO仅需2个（Policy、Ref Model），内存效率提升约50%

6. **GRPO目标函数**：J_GRPO(θ) = E[min(ratio·Â, clip(ratio)·Â)] - β·D_KL(π_θ || π_ref)

7. **实践应用**：GRPO是DeepSeek-R1系列推理模型训练的核心算法，证明了其在超大规模训练中的有效性
