# 4.2 奖励模型训练：排序数据与人类反馈

## 课程概述

本课时深入讲解 RLHF 第二阶段——奖励模型（Reward Model）训练的核心技术。奖励模型是 RLHF 系统的"定海神针"，它将模糊的、主观的人类偏好转化为可微分优化的标量信号，为后续的强化学习策略优化提供监督目标。本课时将从数据收集、模型架构、训练目标三个维度系统解析奖励模型，并探讨标注质量控制、偏好噪声等实际挑战，为理解 DPO 等新型对齐方法奠定基础。

**学习目标**
- 理解奖励模型在 RLHF 中的桥梁作用：连接人类偏好与强化学习优化
- 掌握 Bradley-Terry 偏好模型的理论基础及其在奖励模型训练中的应用
- 理解排序数据的收集流程、标注质量控制与 ELO 评分系统
- 掌握奖励模型的网络结构设计与训练目标函数
- 了解标注不一致性、奖励黑客等挑战及其缓解策略
- 理解 Constitutional AI 与 DPO 如何从不同角度解决奖励模型的局限

**前置知识**：概率论基础（sigmoid 函数、条件概率）、监督学习损失函数、大语言模型 transformer 架构基础

## 1. 奖励模型的角色：人类偏好的可优化代理

### 1.1 为什么需要奖励模型

在 RLHF 第一阶段（SFT）中，模型通过模仿人类专家的演示数据学习"如何回答"。但这种模仿学习存在根本局限：它只能学习"训练数据中的好答案"，无法泛化到未见过的新场景，更无法真正理解"什么是人类认可的好"。

奖励模型的核心价值在于：**它学会了判断质量，而不仅仅是模仿答案**。

```
┌─────────────────────────────────────────────────────────────┐
│           奖励模型在 RLHF 系统中的定位                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Stage 1: SFT Model                                        │
│    输入：提示 x + 专家回答 y                                 │
│    输出：预测下一个词的概率（模仿目标）                        │
│    问题：无法判断回答的绝对质量                               │
│                                                             │
│  Stage 2: Reward Model                                     │
│    输入：提示 x + 回答 y                                     │
│    输出：标量分数 r(x, y)，代表人类偏好程度                    │
│    核心：学会预测"人类会如何评价这个回答"                     │
│                                                             │
│  Stage 3: RL Policy                                         │
│    输入：提示 x                                             │
│    输出：生成回答 y，并通过 max r(x, y) 优化策略               │
│    关键：奖励信号来自 RM，而非固定数据集                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 奖励模型与真实人类偏好的关系

奖励模型是一个代理模型——它学习的是人类偏好的近似表示，而非人类偏好本身。这种代理关系带来了根本性的挑战：

- **代理偏差**：RM 的评分标准可能与真实人类偏好存在系统性偏差
- **泛化局限**：RM 在训练分布外的评分不可信
- **奖励黑客**：Actor 可能学会"欺骗"RM 而非真正满足人类

这意味着奖励模型的质量直接决定了 RLHF 的上限——一个差的 RM 会引导 RL 优化走向错误的方向。

## 2. 排序数据收集：从人类偏好到训练样本

### 2.1 偏好对（Preference Pair）的生成

奖励模型的训练数据是成对的排序标注。每一条训练样本包含：

- 一个提示 x
- 两个候选回答 y_winner（人类偏好的） 和 y_loser（人类不偏好的）
- 标签 label 表示哪个回答更受人类青睐

```
偏好对的数据结构：
┌─────────────────────────────────────────────────────────────┐
│  提示 (x): "解释什么是量子纠缠"                              │
│                                                             │
│  回答 A (y_A): "量子纠缠是量子力学中的一种现象..."            │
│  回答 B (y_B): "量子纠缠就是两个粒子心灵感应..."              │
│                                                             │
│  人类标注: A > B  （A 更准确、更符合物理事实）                  │
│                                                             │
│  训练样本: (x, y_A, y_B, label=1) 表示 A 优于 B              │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 ELO 评分系统：批量质量控制

在实际标注中，大型RLHF项目（如 InstructGPT、ChatGPT）需要数十万对偏好数据，单个标注者的判断可能存在主观偏差。为提高标注质量，研究者引入 ELO 评分系统对标注者进行质量评估：

**ELO 系统的核心思想**：
- 每个标注者有一个隐式的"标注能力分数"
- 当两个标注者对同一问题给出不同判断时，系统根据他们的 ELO 分数推断谁更可能"正确"
- ELO 分数通过迭代更新：预测正确者分数上升，预测错误者分数下降

```
ELO 更新公式：
S_new = S_old + K * (outcome - E[outcome])

其中：
S_old      = 标注者当前 ELO 分数
K          = 学习率（通常 32~64）
outcome    = 实际结果（1 = 判断正确，0 = 判断错误）
E[outcome] = 预期正确率 = 1 / (1 + 10^((R_opponent - R_player) / 400))
```

**在偏好标注中的应用**：
- 同一回答对可能由多个标注者独立判断
- 如果高 ELO 标注者与低 ELO 标注者判断一致，该偏好的可信度更高
- 系统可以根据 ELO 分数对标注数据进行加权筛选

### 2.3 标注质量控制的关键挑战

**挑战一：标注者内部一致性（Intra-rater Agreement）**

同一标注者对相似问题的前后判断可能不一致。这意味着人类的"偏好"本身并非稳定、一致的信号，而是充满噪声的随机变量。

**挑战二：标注者间分歧（Annotator Disagreement）**

当两位标注者给出相反的判断时，系统无法简单判断谁"对"了。常见处理方式：
- 丢弃有分歧的样本
- 使用 ELO 加权，采信高可信度标注者的判断
- 通过多个标注的投票/加权平均获得"群体偏好"

**挑战三：偏见传播（Bias Propagation）**

如果标注者群体存在系统性偏见（如倾向于特定风格或价值观），RM 会学习并放大这些偏见。在 RL 阶段，这些偏见会被进一步强化。

## 3. Bradley-Terry 模型：偏好的概率建模

### 3.1 模型定义

Bradley-Terry 模型是奖励模型训练的理論基础，它将"人类对两个回答的偏好概率"建模为 sigmoid 函数的输出：

```
P(y_winner ≻ y_loser | x) = σ(r(x, y_winner) - r(x, y_loser))

其中：
σ(z)         = 1 / (1 + e^(-z))  是 sigmoid 函数
r(x, y)      = 奖励模型对提示 x 和回答 y 给出的标量分数
```

**核心直觉**：如果奖励模型认为 y_winner 的分数比 y_loser 高出 Δ，那么人类偏好 y_winner 的概率就是 σ(Δ)。

### 3.2 损失函数

给定一组偏好对训练数据，奖励模型的训练目标是最大化人类偏好标注的似然。通过负对数似然推导，得到交叉熵损失函数：

```
L_RM = -E_{(x, y_w, y_l) ~ dataset}[log σ(r(x, y_w) - r(x, y_l))]

或者等价写作：
L_RM = -E_{(x, y_w, y_l, label) ~ dataset}[
    label * log σ(r(x, y_w) - r(x, y_l)) +
    (1 - label) * log σ(r(x, y_l) - r(x, y_w))
]

其中 label = 1 表示 y_w 被偏好，label = 0 表示 y_l 被偏好
```

**简化形式**：
当 label=1 时，损失简化为 `-log σ(r_w - r_l)`，即鼓励 `r_w > r_l`

### 3.3 与语言模型预训练的类比

 Bradley-Terry 损失与标准语言模型损失有本质区别：

| 特性 | 语言模型损失 | Bradley-Terry 损失 |
|------|-------------|-------------------|
| 学习目标 | 最大化 p(y|x) 的似然 | 最大化偏好的正确预测概率 |
| 训练信号 | 确定的（参考答案） | 概率的（偏好关系） |
| 数据结构 | 单回答 | 成对比较 |
| 泛化含义 | 学会"如何说话" | 学会"判断好坏" |

## 4. 奖励模型架构：复用与改造

### 4.1 网络结构

奖励模型通常基于预训练语言模型构建，仅修改输出层：

```
Reward Model = 预训练 LLM backbone + 奖励头（Reward Head）

         ┌──────────────────────────────────────────┐
         │  输入：提示 x + 回答 y                     │
         │          │                               │
         │          ▼                               │
         │  ┌─────────────────────────────────┐    │
         │  │  Transformer Backbone           │    │
         │  │  (与预训练模型共享权重)           │    │
         │  │  理解语言、语义、上下文            │    │
         │  └─────────────────────────────────┘    │
         │          │                               │
         │          ▼                               │
         │  ┌─────────────────────────────────┐    │
         │  │  奖励头（线性层）                  │    │
         │  │  输入：最后一个词元的隐状态        │    │
         │  │  输出：单个标量分数 r(x, y)        │    │
         │  └─────────────────────────────────┘    │
         └──────────────────────────────────────────┘
```

**关键设计点**：
- **骨干网络共享**：使用与 SFT 模型相同的预训练权重，确保 RM 理解语言的能力与 Actor 一致
- **奖励头**：简单的线性变换，将隐状态映射为标量分数，参数量极小
- **输入方式**：将提示 x 和回答 y 拼接为一个序列输入模型

### 4.2 为什么使用最后一个词元

Transformer 的自注意力机制允许每个词元"看到"完整的上下文。奖励模型使用最后一个词元的隐状态作为整体回答质量的表示：

- 最后一个词元已经"阅读"了整个回答
- 它的隐状态编码了从上下文中提取的质量信息
- 线性奖励头从这个状态中提取最终的分数

### 4.3 训练配置与数据规模

**数据规模**：典型的奖励模型训练需要 10K～100K 对偏好数据
- 规模太小：RM 无法学到有意义的偏好模式
- 规模太大：标注成本急剧上升，且边际收益递减

**训练配置**：
- 学习率：通常比 SFT 低（1e-6 ~ 5e-6）
- 训练轮数：1~3 轮（过多会导致过拟合偏好数据的噪声）
- 冻结策略：部分实现冻结骨干网络，仅训练奖励头；更多实践选择端到端训练

## 5. 奖励模型的挑战与应对

### 5.1 标注不一致性问题

不同标注者对同一问题可能给出完全不同的偏好判断，这使得"人类偏好"本身成为一个噪声信号。

**应对策略**：
- **数据过滤**：剔除标注者间一致性低于阈值的样本
- **Elo 加权**：高一致性标注者的判断获得更高权重
- **多元聚合**：使用 Bradley-Terry 模型的变体，对多个标注者的偏好进行概率聚合

### 5.2 奖励黑客（Reward Hacking）

奖励黑客指 Actor 通过发现 RM 的评分漏洞来获取高奖励，而实际人类满意度反而下降的现象。

**典型案例**：
- RM 偏好长回答 → Actor 生成冗长废话但评分高
- RM 偏好特定词汇 → Actor 堆砌关键词刷分
- RM 对某种风格有偏差 → Actor 迎合偏差而非真正有用

**根本原因**：RM 是人类偏好的代理，而非人类偏好本身。当 Actor 探索到 RM 评分边界之外的行为时，RM 的评分变得不可信。

### 5.3 分布外泛化（Out-of-Distribution）

RM 只能对训练数据分布内的回答给出可靠评分。当 RL 阶段的 Actor 生成训练时未见过的回答类型时，RM 的评分可能完全失准。

**应对策略**：
- KL 散度约束：限制 Actor 与参考模型的偏离程度，确保大部分行为在 RM 可靠评分范围内
- RM 定期重训练：用 RL 过程中新生成的数据持续更新 RM
- 保守的 KL 设置：在 reward hacking 风险高的场景中使用更大的 KL 惩罚系数

## 6. 超越标准 RLHF：Constitutional AI 与 DPO

### 6.1 Constitutional AI：AI 反馈替代人类反馈

Anthropic 提出的 Constitutional AI (CAI) 是对 RLHF 的重要改进。CAI 的核心思想是：用人写的"原则"（constitutional principles）替代部分人类偏好标注，通过 AI 反馈训练 Reward Model。

**CAI 的流程**：
1. 使用初始 RLHF 模型生成候选回答
2. 让另一个 AI 模型（Critic）根据原则判断回答的有害性
3. 生成"AI 偏好"数据训练 Reward Model
4. 用 AI 反馈的 RM 进行 RL 训练

```
CAI 的偏好来源：
┌─────────────────────────────────────────────────────────────┐
│  人类偏好数据（部分）                                          │
│    ↓                                                         │
│  "回答不应包含歧视性内容"                                     │
│  "回答应简洁明了"                                             │
│  "回答应避免幻觉"                                             │
│    ↓                                                         │
│  AI 反馈生成偏好对                                            │
│    ↓                                                         │
│  Reward Model 训练                                            │
└─────────────────────────────────────────────────────────────┘
```

**CAI 的优势**：
- 降低人类标注成本
- 减少标注者的主观偏见
- 提高标注一致性

### 6.2 DPO：绕过奖励模型的直接偏好优化

DeepSeek 等团队提出的 DPO（Direct Preference Optimization）提供了一个重要洞察：**奖励模型可能是多余的**。

**DPO 的核心思想**：

传统 RLHF 需要训练奖励模型来提供偏好信号，但奖励模型本身存在代理偏差和泛化问题。DPO 直接在偏好数据上优化策略，绕过奖励模型：

```
标准 RLHF：
  偏好数据 → 训练 RM → 用 RM 的评分信号进行 PPO 优化

DPO：
  偏好数据 → 直接用偏好损失优化策略模型
  L_DPO = -E_{(x, y_w, y_l)}[log σ(
    log π(y_w|x) - log π(y_l|x)  (策略的概率比)
  )]
```

**DPO 的损失函数解读**：

DPO 的损失直接最大化"偏好回答的概率"与"不偏好回答的概率"之比，本质上是在优化策略使得 `π(y_w|x) > π(y_l|x)` 对偏好对成立。这与 Bradley-Terry 模型的精神一致，但不需要显式训练奖励模型。

**DPO 的优势**：
- 省略奖励模型训练步骤，简化 pipeline
- 避免奖励模型的代理偏差
- 训练更稳定，收敛更快

**DPO 的局限**：
- 对数据质量要求更高（偏好标注必须准确）
- 在复杂任务上可能不如 PPO 有效

## 本章小结

1. **奖励模型是 RLHF 的核心**：它将主观、模糊的人类偏好转化为可微分的标量信号，为 RL 策略优化提供监督目标

2. **排序数据是训练基础**：通过人类标注者对候选回答进行成对比较，收集偏好对数据；ELO 评分系统用于标注质量控制

3. **Bradley-Terry 模型提供理论框架**：将偏好概率建模为 sigmoid(r_w - r_l)，训练目标是最小化交叉熵损失

4. **奖励模型架构简洁高效**：复用预训练 LLM 骨干，仅添加线性奖励头；使用最后一个词元的隐状态输出标量分数

5. **标注质量是关键瓶颈**：标注者不一致性、偏好噪声是训练数据的主要挑战；数据规模通常在 10K~100K 对

6. **RLHF 的改进方向**：Constitutional AI 用 AI 反馈补充人类偏好；DPO 绕过高方差、易偏差的奖励模型，直接优化偏好目标
