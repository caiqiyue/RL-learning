# 5.2 量化因子详解：scale/zero_point/校准

## 课程概述

本节课深入探讨大模型量化技术中的三个核心概念：量化因子（scale）、零点（zero_point）和校准（calibration）。我们将详细分析不同粒度的量化因子如何影响量化精度，零点如何确保量化后的数值表示完整性，以及各种校准方法如何帮助确定最优的量化参数。理解这些概念是掌握现代大模型量化技术（如GPTQ、AWQ）的必要基础。

## 学习目标

- 掌握量化因子的三种粒度：按张量、按通道、按token
- 理解零点的作用及其在对称/非对称量化中的意义
- 熟悉四种主要校准方法的原理和适用场景
- 能够根据模型组件类型选择合适的量化配置
- 了解现代大模型量化方法（GPTQ、AWQ）的校准策略

## 先修知识

- 理解浮点数表示（FP32、FP16）基本概念
- 了解整数量化原理（INT8、INT4）
- 熟悉线性量化公式：q = round(x/scale + zero_point)
- 具备基础线性代数知识（张量、矩阵维度）

## 1. 量化因子（Scale）详解

量化因子是连接浮点数世界和整数世界的桥梁，它决定了原始浮点数值与量化整数之间的比例关系。不同粒度的量化因子适用于不同类型的张量，并在精度和开销之间取得平衡。

### 1.1 按张量量化（Per-Tensor Scale）

按张量量化使用单一的scale值覆盖整个张量所有元素。这是最简单的量化方式，整个权重矩阵或激活张量共享同一个缩放因子。

**计算公式：**
```
scale = (x_max - x_min) / (q_max - q_min)
```

**数值示例：**

假设有权重张量 W = [-3.0, -1.5, 0.0, 2.0, 2.5]，目标量化范围为INT8 (-128, 127)：

```
x_max = 2.5, x_min = -3.0
scale = (2.5 - (-3.0)) / (127 - (-128))
      = 5.5 / 255
      ≈ 0.0216
```

量化结果：
- -3.0 → round(-3.0/0.0216) = -139（超出范围，截断为-128）
- 2.5 → round(2.5/0.0216) = 116

**特点：**
- 实现简单，存储开销最小
- 对异常值敏感，可能浪费部分表示范围
- 适合数值分布均匀的张量

### 1.2 按通道量化（Per-Channel Scale）

按通道量化在权重矩阵的每个输出通道上维护独立的scale值。对于Linear层的权重矩阵W[out_features, in_features]，每个out_channel有独立的scale和zero_point。

**为什么对权重使用按通道？**

权重数值分布通常存在通道间不均匀性。某些通道可能包含幅度较大的权重（如attention层的head），而其他通道可能只有微小的权重值。

**数值示例：**

对于权重矩阵 W[3, 4]（3个输出通道，4个输入特征）：

```
通道0权重: [-0.5, 0.3, -0.2, 0.1]
通道1权重: [-2.0, 1.5, -1.8, 1.2]
通道2权重: [-0.1, 0.05, -0.08, 0.02]
```

按通道计算scale（INT8范围[-128, 127]）：

```
通道0: scale_0 = 0.5 / 127 ≈ 0.0039
通道1: scale_1 = 2.0 / 127 ≈ 0.0157
通道2: scale_2 = 0.1 / 127 ≈ 0.0008
```

如果使用按张量量化（使用全局最大绝对值2.0）：
- scale_global = 2.0/127 ≈ 0.0157
- 通道2的0.05将被量化为 round(0.05/0.0157) = 3，反量化后0.047，误差达6%

如果使用按通道量化：
- 通道2的scale_2 = 0.0008
- 同样的0.05量化为 round(0.05/0.0008) = 63，反量化后0.0504，误差仅0.8%

### 1.3 按Token量化（Per-Token Scale）

按token量化是针对激活值的一种粒度，scale基于每个token（样本）独立计算。适用于推理时激活值范围随输入变化的场景。

**工作原理：**

对于输入序列中的每个token位置，计算该位置所有激活值的scale：

```
对于token_i的激活向量 x_i[hidden_dim]

scale_i = max(|x_i|) / (q_max - q_min)
```

**数值示例：**

假设推理时处理两个不同的token，它们的激活值分布不同：

```
Token A的激活: [0.5, 1.2, 0.8, 0.3] → max_abs = 1.2 → scale_A ≈ 0.0094
Token B的激活: [2.0, 3.5, 2.8, 1.9] → max_abs = 3.5 → scale_B ≈ 0.0274
```

注意：按token量化需要动态计算scale，因为激活值在推理时取决于具体输入。

### 1.4 粒度选择指南

| 张量类型 | 推荐粒度 | 原因 |
|---------|---------|------|
| 权重（Weights） | 按通道 | 通道间数值范围差异大，按通道保持精度 |
| 激活值（Activations） | 按张量或按token | 动态范围大，需平衡精度与开销 |
| 嵌入层（Embeddings） | 按张量 | 数值分布相对均匀，动态性低 |

## 2. 零点（Zero Point）详解

零点（zero_point）是量化公式中的偏移量，确保浮点数值0能够被精确表示。这个看似简单的设计有着深远的影响。

### 2.1 为什么需要零点？

神经网络中0是一个特殊值：
- ReLU激活函数将负值变为0
- 注意力机制中的mask矩阵（0表示忽略）
- 稀疏表示中的零元素
- BatchNorm/LayerNorm后的数值中心化

**没有零点的后果：**

假设数据范围[-2.1, 4.7]，使用简单缩放：
```
scale = 6.8/255 ≈ 0.0267
```

量化浮点0：round(0/0.0267) = 0
反量化：0 × 0.0267 = 0（看似正确）

但问题在于，浮点-0.013也会量化为0，无法区分这两个值。这意味着原本应该不同的数值被折叠到同一个整数值，丢失了信息。

### 2.2 非对称量化中的零点

非对称量化公式：
```
q = round((x / scale) + zero_point)
x' = (q - zero_point) × scale
```

零点计算：
```
zero_point = round(q_min - x_min / scale)
```

**数值示例：**

数据范围[-2.1, 4.7]，目标INT8范围[-128, 127]：

```
x_max = 4.7, x_min = -2.1
scale = (4.7 - (-2.1)) / (127 - (-128)) = 6.8/255 ≈ 0.0267

zero_point = round(-128 - (-2.1)/0.0267)
           = round(-128 + 78.65)
           = -49
```

验证关键值：

| 原始浮点 | 量化公式 | 量化结果 | 反量化 | 误差 |
|---------|---------|---------|--------|------|
| -2.1 | round(-2.1/0.0267 - 49) | -128 | (-128+49)×0.0267 = -2.11 | 0.01 |
| 0 | round(0/0.0267 - 49) | -49 | (-49+49)×0.0267 = 0 | 0 |
| 4.7 | round(4.7/0.0267 - 49) | 127 | (127+49)×0.0267 = 4.70 | 0 |

### 2.3 对称量化：零点为零的特殊情况

对称量化是非对称量化的特例，强制zero_point = 0，浮点0映射到整数0。

**适用场景：**

对称量化更适合：
- 权重（通常围绕0对称分布）
- 数值范围已知且对称的数据
- 需要简化计算（避免零点偏移）

**对称vs非对称对比：**

| 特性 | 对称量化 | 非对称量化 |
|------|---------|-----------|
| zero_point | 始终为0 | 需计算 |
| 浮点0映射 | 整数0 | 非零整数（通常） |
| 表示范围 | 对称[-127, 127] | 非对称取决于数据分布 |
| 计算复杂度 | 较低 | 较高 |
| 精度损失 | 对称分布数据更优 | 非对称分布数据更优 |

## 3. 校准（Calibration）详解

校准是确定量化参数（scale和zero_point）的过程。给定一组FP32张量值，校准算法寻找最优的量化参数，使得量化引入的误差最小。

### 3.1 校准的基本概念

**什么是校准？**

校准是使用代表性数据集（calibration dataset）确定量化参数的过程。校准数据集通常是训练数据的一个小子集，用于捕获模型在真实数据上的数值分布特征。

**为什么需要校准？**

- 确定scale需要知道数据的实际范围
- 简单使用全局最大值/最小值对异常值敏感
- 需要在精度和表示范围之间找到平衡

### 3.2 最大值校准（Max Calibration）

最简单的校准方法，使用数据的最大绝对值来确定scale。

**算法：**
```
scale = max(|x|) / (q_max - q_min)
zero_point = 0  # 对称量化
# 或
zero_point = round(q_min - x_min / scale)  # 非对称量化
```

**数值示例：**

数据：[0.3, -0.5, 2.1, -1.8, 0.9, 0.1]

```
max_abs = 2.1
scale = 2.1 / 127 ≈ 0.0165
```

**优点：**
- 实现极其简单
- 计算速度快

**缺点：**
- 对异常值极其敏感
- 如果数据包含少量极端值，scale会被不必要地放大
- 浪费表示范围（大部分值用不到扩展的范围）

**适用场景：**
- 数值分布相对均匀的数据
- 快速原型验证
- 异常值影响可接受的场景

### 3.3 直方图校准（Histogram Calibration）

GPTQ等现代量化方法采用的校准技术，通过分析数值分布的直方图来确定最优scale。

**核心思想：**

不同于简单使用最大值，直方图校准尝试找到使量化误差最小的缩放因子。它会尝试不同的缩放比例，选择那个能最好保留原始分布特性的值。

**算法步骤：**

1. 构建激活值/权重值的直方图
2. 逐步调整scale，寻找最优值
3. 对于每个候选scale，计算量化误差
4. 选择误差最小的scale

**百分位数方法（Max方法变种）：**

使用百分位数而非绝对最大值，减少异常值影响：

```
scale = percentile(|x|, p) / (q_max - q_min)
```

常用的百分位数选择：99%、99.9%、99.99%

### 3.4 熵校准（Entropy Calibration）

GPTQ采用的另一种方法，通过最小化FP32分布与量化后INT分布之间的KL散度来选择scale。

**原理：**

量化本质上是将连续分布映射到离散分布。熵校准试图保持这种映射的信息损失最小。

**KL散度公式：**
```
KL(P || Q) = Σ P(x) × log(P(x) / Q(x))
```

其中P是原始FP32分布，Q是量化后的分布。

**算法：**
1. 将FP32值划分到多个bin（直方图）
2. 对每个候选scale，计算量化后的分布Q
3. 计算KL(P||Q)
4. 选择最小化KL散度的scale

**优点：**
- 综合考虑整个分布，不仅仅是极端值
- 对异常值更鲁棒
- 量化后分布的信息损失最小

**缺点：**
- 计算量较大
- 需要合理的直方图bin数量

### 3.5 移动平均校准（Moving Average Calibration）

主要用于推理阶段动态量化激活值。

**方法：**

不固定scale，而是在推理过程中持续更新：

```
running_max = decay × running_max + (1-decay) × current_max
scale = running_max / (q_max - q_min)
```

**典型参数：**
- decay：0.9-0.99（指数移动平均系数）
- 更新频率：每个batch或固定步数

**适用场景：**
- 激活值量化（随输入变化）
- 在线学习场景
- 输入分布可能漂移的情况

### 3.6 校准方法对比

| 校准方法 | 复杂度 | 精度 | 异常值鲁棒性 | 适用场景 | 代表方法 |
|---------|-------|------|-------------|---------|---------|
| 最大值校准 | O(n) | 中等 | 差 | 快速原型、均匀分布 | PTQ |
| 百分位校准 | O(n) | 中等 | 中等 | 存在少量异常值 | PTQ |
| 直方图校准 | O(n×k) | 高 | 好 | GPTQ/AWQ | GPTQ |
| 熵校准 | O(n×k) | 最高 | 最好 | 精度敏感场景 | GPTQ |
| 移动平均 | O(n) | 动态 | 依赖decay | 推理时激活值 | 动态量化 |

n=数据量，k=搜索尝试次数

## 4. 量化平滑（Smoothing）

### 4.1 问题背景

按通道量化虽然精度高，但不同通道的数值范围可能差异巨大。这导致某些通道的scale很小（精细量化），而另一些通道的scale很大（粗糙量化）。

**问题示例：**

```
通道A: 范围[-0.1, 0.1] → scale = 0.0016 → 量化精度高
通道B: 范围[-10, 10] → scale = 0.158 → 量化精度低
```

这种不均衡会导致某些通道的量化误差显著影响最终结果。

### 4.2 平滑原理

量化平滑（SmoothQuant）的核心思想是将"难量化"通道的缩放因子转移到"易量化"通道。

**数学表达：**

对于权重W和激活X的矩阵乘法 Y = XW^T：

```
Y_ij = Σ_k X_ik × W_jk
```

SmoothQuant引入逐通道缩放矩阵S，使得：

```
Ŵ = W × S  (权重平滑)
X̂ = X × S^(-1)  (激活平滑)
```

平滑后：
```
Y = X̂ × Ŵ^T = (X × S^(-1)) × (W × S)^T = X × W^T = Y
```

数学上等价，但数值分布更利于量化。

### 4.3 平滑效果

| 指标 | 平滑前 | 平滑后 |
|------|-------|-------|
| 权重通道范围差异 | 可能10-100倍 | 通常<10倍 |
| 激活值范围差异 | 动态变化 | 相对稳定 |
| INT4量化精度损失 | 可能>5% | 通常<1% |
| 适用位宽 | 主要INT8 | 可支持INT4 |

## 5. 大模型实际校准方法

### 5.1 GPTQ校准流程

GPTQ（Generative Pre-trained Transformer Quantization）的校准流程：

1. **收集校准数据**：使用数百到数千个token的代表性文本
2. **逐层获取激活/权重分布**： hooks钩住每一层的输入输出
3. **选择校准方法**：默认使用熵校准，也支持直方图校准
4. **计算量化参数**：对每个需要量化的张量独立计算
5. **验证精度**：在验证集上测试量化后的模型质量

### 5.2 AWQ校准流程

AWQ（Activation-aware Weight Quantization）针对激活值而非权重：

1. **分析激活分布**：找出对量化最敏感的激活通道
2. **识别敏感权重**：对应敏感激活的权重需要特殊处理
3. **应用平滑**：将缩放负担从激活转移到不敏感的权重
4. **按通道量化权重**：使用修正后的分布参数

### 5.3 实际配置建议

**LLaMA/GPT类模型推荐配置：**

| 组件 | 权重位宽 | 激活位宽 | 量化粒度 | 校准方法 |
|-----|---------|---------|---------|---------|
| QKV投影 | INT4 | INT8 | 按通道/按token | 熵校准 |
| 前馈网络 | INT4 | INT8 | 按通道 | 熵校准 |
| 嵌入层 | INT8 | - | 按张量 | 最大值校准 |
| LayerNorm | FP16 | FP16 | - | 不量化 |
| 输出层 | INT8 | INT8 | 按张量 | 最大值校准 |

## 总结

本节课我们深入探讨了量化技术中的三个核心概念：

1. **量化因子（Scale）**：连接浮点数和整数的桥梁。按张量、按通道、按token三种粒度分别在精度、开销和适用场景上有所权衡。权重推荐按通道量化，激活值可根据动态性选择按张量或按token。

2. **零点（Zero Point）**：确保浮点0精确映射的关键参数。对称量化（zero_point=0）适合权重，非对称量化适合分布不均的激活值。零点使得非对称数据也能充分利用整数表示范围。

3. **校准（Calibration）**：从FP32张量确定量化参数的过程。最大校准简单但对异常值敏感，熵校准精度最高但计算量大，移动平均校准适合动态场景。现代大模型量化（GPTQ、AWQ）普遍采用熵校准或直方图校准。

4. **量化平滑**：SmoothQuant等技术通过逐通道平滑减少异常通道对量化的影响，使得更低位宽（INT4）的量化成为可能。
